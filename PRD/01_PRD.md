## 1. 제품 개요

### 한 줄 요약
판매자와 구매자를 실시간으로 연결하는 경매 플랫폼으로, WebSocket 기반 실시간 입찰과 Toss Payments 결제까지 완전한 거래 사이클을 제공한다.

### 해결하는 문제
- 기존 경매 플랫폼은 입찰 결과를 페이지 새로고침 없이 확인하기 어렵다
- 낙찰 후 별도 결제 과정이 번거로워 실제 거래 완결율이 낮다
- 모바일에서 실시간 입찰이 불편하다

### 핵심 가치
- **실시간성**: WebSocket STOMP + Redis 분산 락으로 동시 입찰 충돌 없는 정확한 실시간 처리
- **완전한 거래**: 입찰 → 낙찰 → 즉시구매 → 결제까지 하나의 플랫폼에서 완결
- **신뢰성**: 100명 동시 입찰 테스트 완료, Prometheus + Grafana 모니터링

## 2. 사용자

### 주요 사용자

**구매자 (입찰자)**
- **누구**: 물건을 더 싸게 구매하고 싶은 일반 사용자
- **상황**: 경매 종료 전 실시간으로 입찰 경쟁
- **목표**: 원하는 물건을 낮은 가격에 낙찰받아 결제 완료

**판매자**
- **누구**: 물건을 시장 가격 이상으로 판매하고 싶은 사용자
- **상황**: 경매 아이템 등록 후 입찰 현황 모니터링
- **목표**: 최고가 입찰자에게 낙찰, 즉시구매가 설정으로 빠른 판매

### 사용자 시나리오

**구매자 흐름**
1. 사용자가 경매 목록에서 원하는 아이템을 발견하고
2. 실시간 입찰 또는 즉시구매 버튼을 누르면
3. 낙찰 시 Toss Payments로 결제 완료되어 거래가 마무리된다

**판매자 흐름**
1. 판매자가 아이템 등록 (이미지 + 시작가 + 즉시구매가 설정) 하면
2. 경매가 시작되고 입찰자들의 실시간 입찰을 모니터링하며
3. 낙찰 후 결제 완료 알림을 받는다

## 4. 사용자 흐름 (User Flow)

### 입찰 핵심 흐름
```
로그인 → 경매 목록 → 아이템 상세 → 입찰 금액 입력 → 실시간 브로드캐스트 → 낙찰 → 결제
```

### 즉시구매 흐름
```
아이템 상세 → [즉시구매] 버튼 클릭 → 확인 다이얼로그 → 경매 즉시 종료 → 결제 페이지
```

### 결제 흐름
```
낙찰 확인 → [결제하기] 클릭 → Toss Payments 창 → 결제 완료 → 웹훅 처리 → 결제 내역 저장
```

### 상세 흐름

1. **경매 등록**: 판매자가 아이템 정보 + 이미지(S3) + 시작가 + 즉시구매가 + 종료시각 설정
2. **실시간 입찰**: 구매자가 입찰 → Redis 분산 락 → DB 저장 → WebSocket 전체 브로드캐스트
3. **경매 종료**: 스케줄러가 매 1분 만료 경매 처리 → auction_results 생성 → 텔레그램 알림
4. **결제**: 낙찰자 마이페이지에서 Toss Payments로 결제 → 웹훅으로 상태 업데이트

## 5. 성공 기준

- [ ] 100명 동시 입찰 시 단 1건만 낙찰 처리 (Redis 분산 락 검증 완료)
- [ ] 입찰 후 WebSocket 브로드캐스트 지연 < 100ms
- [ ] Toss Payments 결제 성공률 > 99%
- [ ] 모바일(375px) 에서 입찰 및 결제 흐름 완전 동작
- [ ] Grafana 대시보드에서 API p95 응답시간 실시간 확인

---

## 6. 안 만드는 것 (Out of Scope)

> 이 목록에 있는 건 현재 Phase에서 만들지 않습니다.

- **판매자 정산 시스템** — 이유: 실제 비즈니스 복잡성, 포트폴리오 범위 초과
- **경매 라이브 스트리밍** — 이유: 미디어 서버 비용 및 복잡도 과도
- **iOS/Android 네이티브 앱** — 이유: PWA로 모바일 커버리지 충분
- **판매자 리뷰/평점** — 이유: 신뢰 시스템은 운영 정책 필요, 포트폴리오 범위 초과
- **경매 수수료 자동 정산** — 이유: PG사 정산 연동 별도 계약 필요
